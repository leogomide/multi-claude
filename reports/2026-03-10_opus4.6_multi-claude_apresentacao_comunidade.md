## Sumário Executivo

Este relatório consolida uma análise profunda do projeto **multi-claude** realizada por 3 agentes especializados em paralelo, cobrindo arquitetura técnica, experiência do usuário e propostas de casos de uso para apresentação ao vivo em comunidade de desenvolvedores focada em IA.

**multi-claude** é uma CLI tool que transforma o Claude Code de ferramenta single-provider em **plataforma multi-provider**, gerenciando 14 provedores de API (11 cloud + 3 locais), múltiplas contas OAuth, instalações isoladas e monitoramento de custos em tempo real — tudo em uma TUI interativa construída com React para terminal.

## 1. Análise Arquitetural

### 1.1 Visão Geral da Arquitetura

O projeto implementa uma arquitetura **Master-Worker** com separação rigorosa entre o processo principal (`cli.ts`) e a TUI isolada (`tui-process.ts`). Essa decisão garante que o estado do terminal nunca vaze entre sessões.

```
cli.ts (Master Process)
  ├─ Parse CLI args / flags estratégicos
  ├─ Spawn tui-process.ts (Worker Process)
  │   ├─ Load config + migrate installations
  │   ├─ Language selection (first-time)
  │   ├─ Render Ink/React UI (UnifiedApp)
  │   └─ Exit com código de controle
  ├─ Processa resultado via JSON temporário
  ├─ Spawn claude executable com env vars corretas
  └─ Loop: retorna ao TUI após Claude Code sair
```

### 1.2 Exit Codes como Controle de Fluxo

Uma decisão arquitetural elegante: exit codes substituem IPC complexo.

| Code | Significado |
|------|-----------|
| 0 | Sucesso — executa Claude Code |
| 1 | Erro — termina |
| 2 | Restart TUI (ex: idioma alterado) |
| 3 | OAuth login necessário |
| 4 | Update disponível |

A comunicação TUI → CLI acontece via arquivo JSON temporário (`last-selection.json`), que é lido pelo master e deletado após processamento. Essa abordagem é mais debugável que sockets/pipes e resiliente a crashes.

### 1.3 Padrões de Design Identificados

| Padrão | Localização | Benefício |
|--------|------------|----------|
| **Strategy** | `ProviderTemplate` + `configureEnv()` | Cada provider customiza env vars sem alterar código core |
| **Adapter** | `api-models.ts` | Normaliza 6 APIs diferentes em `ApiModelMeta` unificado |
| **Provider (React Context)** | `I18nProvider`, `BreadcrumbProvider` | Dependency injection sem prop drilling |
| **Factory** | `getTemplate(id)` | Lookup seguro de providers por ID |
| **Master-Worker** | `cli.ts` + `tui-process.ts` | Isolamento de estado terminal |
| **State Machine** | `StartClaudeFlow` steps | Fluxos de usuário previsíveis |

### 1.4 Stack Tecnológico — Decisões e Justificativas

| Tecnologia | Por que? |
|-----------|---------|
| **Bun** | Roda `.ts` direto sem build step, startup 3-5x mais rápido que Node.js |
| **Ink (React)** | UI declarativa para terminal com composição de componentes e hooks |
| **Zod** | Validação runtime + type inference (`z.infer<>`) — single source of truth para types |
| **Rosetta** | i18n leve, type-safe, com interpolação |

### 1.6 Sistema de Providers — Extensibilidade

Adicionar um novo provider requer apenas ~10 linhas em `providers.ts`:

```typescript
{
  id: "novo-provider",
  description: "Novo Provider",
  baseUrl: "https://api.novo.com/anthropic",
  defaultModels: ["model-a", "model-b"],
  env: {},
}
```

O `buildClaudeEnv()` faz o resto: limpa variáveis conflitantes, aplica env vars do template, define modelo em 4 variáveis de compatibilidade.

### 1.7 Segurança e Credenciais

- **OAuth**: Tokens armazenados em `~/.multi-claude/accounts/{providerId}/.credentials.json` com validação de expiração (5 min buffer)
- **API Keys**: Salvas em `config.json` (plain-text) e passadas exclusivamente via env vars ao spawn
- **Isolamento**: Cada instalação tem seu próprio `CLAUDE_CONFIG_DIR`, impedindo vazamento de dados entre contextos

### 2.2 Fluxos Principais de Usuário

#### Fluxo 1: Primeiro Uso
```
mclaude → Seleção de idioma → Menu Principal → Add Provider
→ Selecionar template → Nome → API Key → Modelos → Salvar
→ Start Claude → Selecionar provider → Modelo → Launch
```

#### Fluxo 2: Uso Cotidiano
```
mclaude → Menu Principal → Start Claude
→ Provider (pré-selecionado) → Modelo → Flags/Opções → Launch
→ Claude Code executa → Volta ao menu (loop)
```

#### Fluxo 3: OAuth (Anthropic)
```
Add Provider → Template "Anthropic" → Authenticate
→ Claude Code abre login nativo → .credentials.json salvo
→ Próximas sessões: token automático
```

#### Fluxo 4: Headless (Automação)
```bash
mclaude --provider deepseek --model deepseek-chat -p "query"
mclaude --list  # JSON com todos providers/modelos/instalações
```

### 2.4 Sistema de Instalações

Instalações são diretórios isolados de configuração Claude Code:

```
~/.multi-claude/installations/
├── abc123-projeto-a/     # Projeto A: MCP servers, histórico, agents próprios
├── def456-cliente-b/     # Cliente B: contexto isolado
└── ghi789-pessoal/       # Pessoal: configurações livres
```

**Regras de auto-seleção:**
- API providers sem instalações custom → usa Default (`~/.claude/`)
- API providers com >1 instalação → usuário escolhe
- OAuth Anthropic → **obrigatório** usar instalação custom (isolamento de credenciais)

### 2.6 Flags e Opções de Launch

5 grupos configuráveis no momento do launch:

| Grupo | Flag | Efeito |
|-------|------|--------|
| Session | `--resume` | Continuar conversa anterior |
| Permissions | `--dangerously-skip-permissions` | Bypass de confirmações |

## 3. Ecossistema e Proposta de Valor

### 3.1 Problema Central

Claude Code nativamente suporta apenas 1 provider por sessão. Alternar exige editar variáveis de ambiente manualmente, reconfigurar endpoints e gerenciar múltiplos scripts. **multi-claude** resolve isso com uma interface unificada.

```
Antes:
  export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
  export ANTHROPIC_AUTH_TOKEN=sk-xxx
  export ANTHROPIC_MODEL=deepseek-chat
  claude -p "query"

Depois:
  mclaude --provider deepseek -p "query"
```

### 3.2 Tabela de Economia de Custos

| Cenário | Provider | Custo/MTok (Input) | Economia vs Claude Opus |
|---------|----------|-------------------|------------------------|
| Máximo poder | Claude Opus | $15.00 | — |
| Balanceado | Claude Sonnet | $3.00 | 80% |
| Ultra-econômico | DeepSeek Chat | $0.55 | **96%** |
| Agregador smart | OpenRouter (best price) | $0.50-$2.00 | 87-97% |
| Local/offline | Ollama/llama.cpp | $0.00 | **100%** |

### 3.3 Posicionamento no Ecossistema

```
Claude Code (Anthropic)
    │
    ├── Uso direto: 1 provider, config manual
    │
    └── via multi-claude:
        ├── 14 providers configuráveis
        ├── Troca instantânea entre providers
        ├── Instalações isoladas por contexto
        ├── Status line com custo/performance
        ├── Headless mode para automação
        └── i18n (3 idiomas)
```

## 4. Casos de Uso para Apresentação na Live

### Caso 1: "O Dev Indie Otimizado" — Economia de 98%

**Cenário:** Desenvolvedor indie com orçamento limitado precisa de IA para codificar diariamente.

**Demo ao vivo:**
1. `mclaude` → TUI mostra 14 provedores disponíveis
2. Seleciona Ollama (local) → gera código com custo zero
3. Troca para DeepSeek → review com custo de $0.02
4. Status line mostra custo total da sessão em tempo real

**Impacto:** Economia de ~$50/dia → **$15k/ano** em custos de desenvolvimento com IA.

---

### Caso 2: "O Consultor Multi-Cliente" — Isolamento Total

**Cenário:** Consultor atende 5 clientes simultaneamente, cada um exigindo isolamento de dados.

**Demo ao vivo:**
1. 3 contas Anthropic OAuth configuradas (pessoal, ClienteA, ClienteB)
2. Seleciona "ClienteA" → instalação isolada com histórico próprio
3. Muda para "ClienteB" → contexto completamente separado
4. MCP servers, agents e settings diferentes por instalação

**Impacto:** Compliance total, zero vazamento de dados entre contextos.

---

### Caso 3: "O Benchmark de Modelos" — Comparação Visual

**Cenário:** Engenheiro ML quer escolher o melhor custo-benefício entre modelos.

**Demo ao vivo (3 terminais lado a lado):**
```
Terminal 1: mclaude --provider anthropic --model claude-opus-4  → $1.50
Terminal 2: mclaude --provider deepseek --model deepseek-chat    → $0.02
Terminal 3: mclaude --provider ollama --model llama2              → $0.00
```
Status lines em tempo real comparando latência, tokens e custo.

**Impacto:** Decisão informada em 10 minutos, potencialmente **$10k/ano de economia**.

### Caso 4: "Dev Offline-First" — IA Sem Internet

**Cenário:** Desenvolvedor em área remota ou com requisitos de privacidade.

**Demo ao vivo:**
1. Desconecta WiFi (dramaticamente)

**Impacto:** Desenvolvimento com IA em qualquer lugar, privacidade total (LGPD/GDPR).
